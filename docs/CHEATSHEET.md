# 🚀 開発用サーバー操作・解説チートシート (req-assistant)

このドキュメントは、`req-assistant` (VibeArchitect) の開発プロジェクトで使用する major なコマンドとその解説をまとめたものです。

---

## 🛠️ 環境準備 (First Setup)

開発を始める前に、一度だけ実行が必要な操作です。

| コマンド               | 解説                                                                       |
| :--------------------- | :------------------------------------------------------------------------- |
| `npm install`          | プロジェクトに必要なライブラリ（node_modules）をすべてインストールします。 |
| `cp .env.example .env` | 環境変数のテンプレートをコピーして設定ファイルを作成します。               |

> [!IMPORTANT]
> **APIキーの設定:** `.env` ファイルを開き、`VITE_GEMINI_API_KEY` に自身の Google Gemini API キーを設定してください。設定しない場合、AI機能が動作しません（モックモードでの動作になります）。

---

## 💻 サーバーのライフサイクル (起動・停止・確認)

### 1. サーバーを起動する (Start)

```bash
npm run dev
```

### 2. 稼働状況を確認する (Check)

- **ターミナル:** `Local: http://localhost:5173/` が表示されていれば稼働中です。
- **ブラウザ:** `http://localhost:5173/` を開き、画面が表示されれば正常です。

### 3. サーバーを停止する (Stop)

ターミナル上で **`Ctrl + C`** を押し、`y` を入力して `Enter` で停止します。

---

## 👻 バックグラウンドで動き続けている場合 (Force Kill)

「ターミナルを閉じたのにポートが使われている」「サーバーを止めたはずなのにブラウザがまだ生きている」といった場合の対処法です。

### 1. ポート（5173）を使っているプロセスID (PID) を探す

PowerShellで以下のコマンドを実行します：

```powershell
netstat -ano | findstr :5173
```

実行すると、右端に数字（PID）が表示されます：

```text
  TCP    0.0.0.0:5173           0.0.0.0:0              LISTENING       12345
  TCP    [::]:5173              [::]:0                 LISTENING       12345
```

この例では `12345` が **PID** です。

### 2. そのプロセスを強制終了する

見つかった PID を使って以下のコマンドを実行します（例として `12345` を使用）：

```powershell
taskkill /F /PID 12345
```

これで、バックグラウンドに残っていたサーバーが完全に停止します。

## 🛠️ その他のコマンド (npm scripts)

| コマンド          | 解説                 | 用途                                          |
| :---------------- | :------------------- | :-------------------------------------------- |
| `npm run build`   | プロダクションビルド | 本番公開用のファイルを `dist/` に生成します。 |
| `npm run preview` | ビルド内容の確認     | `build` 後の動作をローカルで最終確認します。  |
| `npm run lint`    | コード解析 (ESLint)  | コードの書き方の間違いをチェックします。      |

## 💡 トラブルシューティング & Tips

- **ポート番号が既に使用されている:**
  Vite はポート `5173` が埋まっている場合、自動的に `5174` などの空きポートを使います。
- **APIキーが反映されない:**
  `.env` を変更した後は、必ず一度停止（Ctrl+C）して再起動してください。
- **ショートカットキー:**
  サーバー稼働中にターミナルで **`h`** キーを押すと、利用可能な全ショートカットが表示されます。
