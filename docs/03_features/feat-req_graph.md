# アプリ設計Platform 運用ルール (フェーズ1: 小規模・非エンジニア向け)

## 1. 運用方針 (BP)

- 対象ユーザー: 非エンジニア
- 想定規模: 小規模アプリ
- コアフロー: 本Platform内で設計構築を完結させ、「要件定義 ＋ Mermaid」を統合したテキストを出力。ユーザーはそれをコピーし、AIエージェントへの指示（MP）としてそのままペーストして使用する。

## 2. MP（出力テキスト）の構成ルール

小規模開発におけるAIのコンテキスト理解を最大化するため、指示出しの分割は行わず、以下の要素を1つのMPとして統合出力する。

1. アプリの概要と目的（要件定義）
2. 画面遷移図（Mermaid: UI/UXの定義）
3. ER図（Mermaid: データ構造の定義）
4. 開発にあたっての制約事項や技術スタック

## 3. 整合性・手戻り防止の仕組み

非エンジニアの目視によるエラーや転記ミスを防ぐため、以下の設計ルールを適用する。

- プレビューの必須化: Platform上でMermaidのコードだけでなく、必ずレンダリングされた図をプレビュー表示し、ユーザーが視覚的に「画面とデータの辻褄が合っているか」を確認できる導線を作る。
- 単一コピーの徹底: ユーザーがテキストの一部だけをコピーし忘れることを防ぐため、最終的な出力テキストは「コピーボタン」を一つ押すだけでMP全体をクリップボードに取得できるUIとする。

## 4. 将来の拡張性（中長期的なWFへの備え）

- 現状は手動コピーを正とするが、将来的にデータや機能が肥大化し、単一のMPではAIが処理しきれなくなった場合に備え、「UI実装用」「DB構築用」など、MPを機能単位で分割出力できる拡張の余地を内部的に持たせておく。
