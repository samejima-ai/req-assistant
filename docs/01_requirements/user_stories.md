# ユーザーストーリー (User Stories)

本プラットフォーム内で、ユーザーがシステムを通じて「何を達成したいか」「どのような体験を得るか」を定義します。

## コア体験 (UX)

**左ペイン（対話） × 右ペイン（視覚化）**の分割インターフェース。
ユーザーはコンサルタントAIと対話するだけで、背後でAIがスキルを駆使して要件を解析し、右側のホワイトボード上に視覚的な設計図（付箋、ワイヤーフレーム、フロー図）をリアルタイムに構築します。

---

## 1. 対話型要件定義 (CaaF)

- **要望の壁打ち**: ユーザーは、曖昧な要望や現場の課題をコンサルタントAIと対話しながら深掘りし、言語化することができる。
- **AIからの能動的な提案と逆質問**: 入力情報からシステム的に不足している要素（エラー処理やデータ保持の必要性など）をAIが自動検知し、AIからの逆質問に答えるだけで要件の漏れを埋めることができる。

## 2. 動的ホワイトボードマッピング

- **チャットからの自動図解生成**: チャットで話した文脈から重要な要素（アクター、アクション、データ）が抽出され、キャンバス上に視覚的ノードとして自動配置されるのを見ることができる。
- **解像度の段階的向上**: CDD（Context-Driven Design）のアプローチにより、最初は抽象的なマインドマップだったものが、対話を通じて具体的な画面構成図・データ関係図へとシームレスに進化していくのを体験できる。

## 3. スコープ・軌道修正制御

- **要件肥大化のリスク回避**: AIが要件の肥大化や論理的破綻を常時監視してアラートを出してくれるため、プロジェクトが炎上するのを防ぐことができる。
- **優先順位の強制仕分け**: 機能をMust/Should/Couldへと強制的に仕分けさせられ、今すぐ不要な機能は「フェーズ2以降」として視覚的に別エリアへ退避されることで、初期リリースのスコープを適切に保つことができる。

## 4. AI開発向けエクスポート

- **AIエージェントへのシームレスな引き継ぎ**: 確定した設計データを、後続のAIコーディングエージェントに直接渡せる形式（構造化されたプロンプト文、ドキュメントなど）として出力でき、「バイブコーディング」による非効率な再作成作業なしに、最高速度での初期構築を開始できる。
